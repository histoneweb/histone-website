## Development Guidelines

### Code Structure
```
/src
  /components
    /common (Header, Footer, ThemeSwitcher)
    /sections (Hero, Portfolio, Expertise)
    /ui (Buttons, Cards, Forms)
  /styles
    /themes (dark.css, light.css)
    /components
  /lib
    /utils
    /constants
  /data
    /projects.json
    /clients.json
    /expertise.json
```

### Component Patterns
- Use composition over inheritance
- Implement lazy loading for images
- Create reusable card components
- Maintain consistent spacing system
- Use CSS modules or styled-components

### State Management
- Theme state (Context API or Zustand)
- Form state (React Hook Form)
- Analytics tracking (Google Analytics 4)
